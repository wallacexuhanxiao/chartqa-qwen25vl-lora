# ChartQA Qwen2.5-VL-7B LoRA Fine-tuning

This repository contains the core code and reproducibility materials for a ChartQA visual question answering project using Qwen2.5-VL-7B-Instruct with LoRA fine-tuning via LLaMA-Factory.

The project builds an end-to-end pipeline for:

- converting HuggingFaceM4/ChartQA into multimodal ShareGPT-style JSONL,
- training Qwen2.5-VL LoRA adapters for short-answer chart QA,
- running baseline and LoRA inference,
- evaluating exact match and relaxed numerical accuracy,
- analyzing failure cases by error type.

## Key Result

On the ChartQA test split, full-data LoRA improved Exact Match from **71.12%** to **78.00%** compared with the zero-shot baseline, while maintaining strong relaxed accuracy.

| Run | N | Exact Match | Relaxed Acc | Numeric Relaxed Acc | Nonempty |
|---|---:|---:|---:|---:|---:|
| base_test | 2500 | 0.7112 | 0.8848 | 0.8928 | 1.0000 |
| lora_full_test | 2500 | 0.7800 | 0.8892 | 0.8944 | 1.0000 |

## What Is Not Included

This repository intentionally excludes large artifacts:

- Qwen2.5-VL-7B base model weights,
- LoRA adapter weights,
- full ChartQA images and JSONL files,
- full prediction JSONL and training logs.

These can be regenerated by following `docs/GUIDE.md`. The GitHub repository keeps sample JSONL files but does not store sample images.

## Minimal Usage

Prepare data:

```bash
python scripts/prepare_chartqa_hfm4.py --out_dir data/chartqa/processed/hfm4
```

Run baseline inference:

```bash
PYTHONPATH=. python eval/infer_chartqa_qwen25vl.py \
  --model_path models/Qwen2.5-VL-7B-Instruct \
  --input_jsonl external/LLaMA-Factory/data/chartqa_test.jsonl \
  --output_jsonl outputs/chartqa/predictions/base_test_full.jsonl \
  --max_new_tokens 32 \
  --attn sdpa
```

Evaluate:

```bash
python eval/eval_chartqa.py \
  --pred_jsonl outputs/chartqa/predictions/base_test_full.jsonl \
  --out_metrics outputs/chartqa/metrics/base_test_full_metrics.json
```

See `docs/GUIDE.md` for the full workflow.
